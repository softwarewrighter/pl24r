Implement the core linker in src/linker.rs that merges multiple parsed .spc modules into one combined .spc text output. The linker must: (1) order modules correctly — runtime modules first, then user libraries, then app module last so main is the entry point, (2) strip .module/.endmodule/.export/.extern directives from output since they are linker-only metadata, (3) merge all .global declarations into one section at the top, (4) merge all .data declarations, (5) merge all .const declarations (checking for conflicts), (6) concatenate all .proc/.end blocks preserving internal labels, (7) preserve comments for debuggability, (8) ensure the output is valid standalone .spc that pasm can assemble directly. The emitter in src/emitter.rs should produce clean, well-formatted .spc text output. Test: cargo test with multi-module merge scenarios, verify output parses back cleanly, verify correct ordering. cargo clippy -- -D warnings must pass.
