Build the symbol table and validation layer in src/symbols.rs. Collect all symbols from parsed .spc modules: procedure names (from .proc), global variables (from .global), data blocks (from .data), constants (from .const). For each module, track which symbols are exported (.export directives or all symbols in export-all fallback mode) and which external symbols are required (.extern directives). Build a global symbol table across all input modules. Implement validation: (1) detect duplicate exported symbols across modules with clear error messages naming both modules, (2) detect unresolved externs — symbols declared .extern but not exported by any module, (3) verify exactly one module contains a main proc (the entry point), (4) warn on unused exports (exported but never referenced as extern). Test: cargo test with cases for valid multi-module resolution, duplicate detection, unresolved extern detection, missing main detection, and export-all fallback. cargo clippy -- -D warnings must pass.
